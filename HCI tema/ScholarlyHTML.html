1. Introduction

1.1 Scope

The scope of this document is to present and explain certain characteristics of the "Learn Dafny" website. These will be the application's purpose, it's User Interface and the way different users can interact with it.

1.2 Document Conventions

This document was created using the "IEEE for System Requirement Specification Documents" template and the "Scholarly HTML" specifications.

2. General Description

2.1 Product perspective

"Learn Dafny" is a website built for everyone that wants to learn and experience a new programming language and also be able to save their progress and all of their projects in a single place by creating an account.

2.2 Product functions

Compiler Page

  • Edit current file
  • Switch to a different file
  • Save program on your account's files
  • Save program on your local computer
  • Create a new program file
  • Run and compile current file
  • Check Dafny documentation

Log In / Sign Up Page

  • Log into an account using a username and a password
  • Create a new account and log in
  • Change the password using an email associated with an account

My files Page

  • See all of the saved files
  • Create a new empty file

Settings Page

  • Change username
  • Change email
  • Change password

2.3 User Classes and Characteristics

  • Users that have little to no programming experience and want to start out by learning Dafny
  • Users that already have some programming skills and want to expand their knowledge
  • Users that have worked with Dafny before and want to learn something new or just want a new working environment

3. Application Architecture

3.1 Log In / Sign Up Page

The first page that opens up when an user acceses the site is the "Log In/Sign Up" page where they will be asked to either log into an existing account using a username and the respective password or create a new account using the former mentioned attributes and an email.

3.2 Editor/Compiler Page

After the user is logged in, the next page that opens up is the editor/compiler one. If the user has just created a new account then the editor side of the page will display a blank file. If the user has previously worked on something then that file will be on display.

Here the user can run and compile a file they choose and get the results on the "OUTPUT" side of the page (like in the example presented in the picture). They do this by clicking the button on the left side. Here they can also check the official documentation for Dafny (the button on the right side).

After they are done with working on their program or want to save their work there is a dropdown button on the header named "File" which has the options to save their work on their account, on their personal computer, to delete the current file and also to load another file from their computer onto the website editor. This can also be done with the "+" button next to the files opened in the editor.

Another thing a user can do is check the examples provided by the website by clicking the dropdown button named "Examples" that is next to the "File" one and further develop them.

3.1 My files Page

From the editor page the user can click on their username at the top right side of the page. The next page that opens will be the My files page where they can see all the files they've saved. Here they can open one in the editor, create a new one or load a file from their computer similar to the dropdown "File" from the editor page.

Here they could also choose to go to their settings or log out.

3.1 Settings Page

Coming from the files page, here the user can change anything about their profile: profile picture, username, email and password.

Again they could also choose to go to their files page or log out.

3. Concepts

3.1 Design Patterns

  • Syntax Highlighting - In the editor page, writing code with little to no syntax highlighting is really difficult and after some time it could cause eyestrain
  • Line Numbering - In the editor page the code lines are numbered making searching for errors and bugs easier
  • Navigation - In the files page the user can search for a file using the search bar
  • Keyboard Shortcuts - At the top of the "Run" button the shortcut for the same function is written because the website contains repetitive tasks that involves switching modes from keyboard to mouse (writing code to running it)
  • Autosaving - At the top of the files header there is specified that the files are autosaved every 30 seconds and that helps the user focus on producing great content without worrying about whether their data is safe and saved

4. Questions, Options, Criteria

4.1 Q: What happens if the website is used from a mobile?

Because on the larger screens in the main page the editor part and the output/terminal part are side by side on a smaller screen they will be stacked upon one another.

4.2 Q: What if the user has too many files saved on his account and can't find one?

The search bar was created for this and also in the example picture it is visible that in the rectangle associated with file_1 you can see a portion of the code saved in that same file. That is because the user may remember what the file contained just from the snapshot of the code.

4.3 Q: What if the user can't see the code they are writing clearly?

For this potential issue the editor uses syntax highlighting and also line numbering. Besides, they may also zoom in on the code using zoom in and zoom out icons on the right side of the editor.
